## 场景描述

你有没有过这种经历？

让AI帮你改一个大型项目里的bug，结果它先花了20分钟翻文件。

grep搜索→读文件→再搜索→再读更多文件，来回折腾十几轮，token 哗哗地烧。

最后还可能找错了调用链。

小项目还好，文件少，AI很快能上手。

但一到几万、几十万行的老项目，或者微服务架构的项目，AI就像进了迷宫。

每次对话都要重新摸路。

**问题出在哪？** 因为AI没有"记忆"。每次新会话，它都从零开始理解你的代码库。

今天给大家推荐一个GitHub近期爆火的开源项目——`codebase-memory-mcp`。

它能给你的AI编程助手装上一个"代码库记忆大脑"，让结构查询的 token 消耗直接降低约 120 倍。

## 操作步骤

**第一步：安装**

支持 macOS、Linux、Windows，一行命令搞定。

**macOS / Linux（普通版）：**

```bash
curl -fsSL https://raw.githubusercontent.com/DeusData/codebase-memory-mcp/main/install.sh | bash
```

**macOS / Linux（带3D可视化UI版）：**

```bash
curl -fsSL https://raw.githubusercontent.com/DeusData/codebase-memory-mcp/main/install.sh | bash -s -- --ui
```

**Windows PowerShell：**

```powershell
Invoke-WebRequest -Uri https://raw.githubusercontent.com/DeusData/codebase-memory-mcp/main/install.ps1 -OutFile install.ps1
.\install.ps1
```

> 小贴士：安装脚本会自动检测你电脑上已安装的AI编程工具（Claude Code、Cursor、VS Code等43种），并自动配置好MCP连接，不用手动改配置。

**第二步：验证安装**

安装完成后，在终端输入：

```bash
codebase-memory-mcp --version
```

如果能输出版本号，说明安装成功。

如果提示"command not found"，手动加一下PATH：

```bash
# macOS / Linux
export PATH="$HOME/.local/bin:$PATH"
```

**第三步：给项目建索引**

进入你的项目目录，然后直接跟AI助手说：

> "Index this project"

或者用CLI手动索引（推荐用绝对路径）：

```bash
codebase-memory-mcp cli index_repository '{"repo_path": "/absolute/path/to/your/project"}'
```

索引速度参考：

| 项目规模 | 索引时间 |
|---|---|
| 普通中型项目（几万行） | 几秒钟 |
| Django（4.9万节点） | 约 6 秒 |
| Linux内核（2800万行） | 约 3 分钟 |

索引完成后，知识图谱会自动保存在本地，下次不用重建。

**第四步：开始使用**

索引建好后，你就可以用自然语言问AI各种结构问题了。

举几个实用例子：

1. **查调用链**："帮我查一下 `UserService.createUser` 被哪些地方调用，调用链是什么？"
2. **架构概览**："分析一下这个项目的整体架构，入口和核心模块有哪些？"
3. **影响分析**："我准备修改 `OrderController.submitOrder`，可能影响到哪些模块？"
4. **死代码检测**："找出项目中可能没有调用者的函数，按风险排序。"
5. **语义搜索**："找一下项目里和重试、退避相关的代码。"

## 进阶玩法

**1. 3D图谱可视化**

如果你装了UI版，可以启动可视化界面：

```bash
codebase-memory-mcp --ui=true --port=9749
```

然后浏览器打开 `http://localhost:9749`，就能看到整个项目的3D知识图谱。

节点是函数/类，连线是调用关系，非常直观。

**2. 自动同步索引**

代码改动后不用手动重建索引，开启自动同步即可：

```bash
codebase-memory-mcp config set auto_index true
```

背景监控器会检测文件变化，增量更新图谱。

**3. 团队共享图谱**

可以把索引快照提交到Git仓库，同事拉下来直接用，跳过全量索引：

```text
.codebase-memory/graph.db.zst
```

**4. 排除不需要索引的目录**

在项目根目录建一个 `.cbmignore` 文件，写法和 `.gitignore` 一样：

```text
node_modules/
dist/
build/
coverage/
```

**5. 支持的客户端**

目前支持 43 种AI编程客户端，包括：
- Claude Code、Codex CLI、Gemini CLI
- Cursor、VS Code、Windsurf
- Aider、OpenHands、Cline
- 以及国内的 Kimi Code、通义灵码等

> 基本上你在用的主流AI编程工具，它都支持。
