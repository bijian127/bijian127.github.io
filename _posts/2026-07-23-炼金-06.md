你的 Prompt 越长，AI 越容易"串台"。

指令、背景资料、示例数据全塞进一段话里，AI 分不清哪句是要求、哪句是素材，输出质量直线下降。

今天分享一个 2026 年依然被严重低估的技巧：**用 XML 标签给 Prompt 分层**。不用学编程，不用懂前端，只需会打 `<` 和 `>` 两个符号。

## 你的 Prompt 是不是越长越乱？

看一个典型场景：你让 AI 帮你改一封客户邮件，同时给了品牌调性说明和原文。

普通写法——所有信息混在一起：

```markdown
你是一个专业的客服。请根据我们的品牌调性重写这封邮件。

我们的品牌调性：自信但不傲慢，不使用陈词滥调，
每封邮件不超过 80 字。

以下是客户邮件原文：
"你们的产品又出问题了，我已经等了三天
还没收到退款，非常失望。"

请重写。
```

AI 可能会：

- 忽略 80 字限制
- 把品牌调性混进回复内容里
- 语气把握不准，要么太冷淡要么太谄媚

问题出在哪？**AI 无法区分"指令"和"素材"**。

它看到一堆文字，只能靠猜测判断哪些是规则、哪些是数据、哪些是待处理内容。信息越多，猜错的概率越高。

## 用 XML 标签分层，效果立竿见影

同样的任务，加几对 XML 标签：

```xml
<role>你是一个专业的客服人员</role>

<task>根据品牌调性重写客户邮件</task>

<guidelines>
- 自信但不傲慢
- 不使用陈词滥调
- 邮件正文不超过 80 字
</guidelines>

<email>
你们的产品又出问题了，我已经等了三天
还没收到退款，非常失望。
</email>

<output_format>
直接输出重写后的邮件，不要解释。
</output_format>
```

**使用 XML 标签前后对比**：

| 对比项 | 使用前 | 使用后 |
|--------|--------|--------|
| 信息分层 | 指令和数据混在一起 | 每类信息独立隔离 |
| AI 理解准确度 | 靠猜测区分素材和指令 | 标签明确标注信息类型 |
| 输出一致性 | 每次风格可能不同 | 格式和语气稳定可复现 |
| 可维护性 | 改一个条件要重写整段 | 修改对应标签即可 |
| 最终状态 | 能用，但不可靠 | 稳定、可复用、可迭代 |

## 为什么 XML 标签这么管用？

原理不复杂，核心就三点：

**消除歧义。** 没有标签时，AI 要自己判断"品牌调性"是待处理内容还是参考规则。加了 `<guidelines>` 标签，AI 立刻知道这是约束条件，不是素材。

**降低认知负担。** 据 AI 培训机构 Future Factors 的总结，结构化 Prompt 比平铺式的可靠性提升 35% 到 95%（取决于任务复杂度）。

标签本质上是帮 AI 做"信息分类"，减少它的判断工作量。信息分层越清晰，AI 的输出越稳定。

**可复用、可协作。** 标签化的 Prompt 像乐高积木。换一封客户邮件？只改 `<email>` 里的内容。换一套品牌调性？只改 `<guidelines>`。

团队成员拿到你的模板，一眼就能看懂哪里填什么。日常反复使用的 Prompt，值得花两分钟用标签整理一下。

## 三个场景，直接套用

**场景一：周报生成**

```xml
<role>你是一位资深项目经理</role>

<task>根据本周工作记录生成周报</task>

<format>
1. 本周完成（3-5 条，每条一行）
2. 风险与阻塞（如有）
3. 下周计划（3 条）
字数不超过 400 字。
</format>

<work_log>
周一：完成产品需求文档初稿
周二：与设计团队对齐交互方案
周三：修复了 3 个线上 bug
周四：参加部门季度复盘会
周五：整理用户反馈并分类
</work_log>
```

**场景二：会议纪要整理**

```xml
<role>你是一位专业的会议记录员</role>

<task>将会议转写文本整理为结构化纪要</task>

<rules>
- 每条决议标注负责人和截止日期
- 待办事项用"□"标记
- 省略寒暄和重复发言
</rules>

<transcript>
（粘贴会议转写文本）
</transcript>
```

**场景三：竞品分析**

```xml
<role>你是一位产品经理</role>

<task>对比两款产品的核心功能差异</task>

<output_format>
用表格输出，列：功能、产品A、产品B、差异评价。
</output_format>

<product_a>
产品名称：
核心功能：
目标用户：
</product_a>

<product_b>
产品名称：
核心功能：
目标用户：
</product_b>
```

## 标签起名建议

不用想太多，标签名能"望文生义"就行：

| 标签名 | 用途 |
|--------|------|
| `<role>` | 定义 AI 角色 |
| `<task>` | 描述具体任务 |
| `<context>` | 提供背景信息 |
| `<guidelines>` | 列出规则和约束 |
| `<data>` | 放入待处理的数据 |
| `<output_format>` | 指定输出格式 |
| `<example>` | 给出示例 |
| `<tone>` | 规定语气风格 |

> 标签名用英文或中文都可以，AI 都能理解。关键是保持一致——一个 Prompt 里别混用中英文标签名。

**XML 标签不是魔法，但它是 2026 年性价比最高的 Prompt 优化手段之一。** 花几秒钟加几对标签，换回更稳定、更可控的 AI 输出。

下次你的 Prompt 超过 5 行，试试用标签分个层。
