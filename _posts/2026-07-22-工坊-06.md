**场景**：你用 Claude Code / Cursor 写代码，AI 确实能写了，但写出来的代码"能用"却"不工程"——没测试、没设计、逻辑混乱。你得花更多时间去修 AI 写的代码，比从头写还累。

**效果预览**：
- Before：AI 写一坨代码，你自己写测试、做重构、查逻辑，修代码的时间比写代码还长
- After：AI 先写测试、再写实现，自动做设计评审，代码质量接近资深工程师水准

## 这个工具是什么

[**obra/superpowers**](https://github.com/obra/superpowers) ⭐ 24 万+

Superpowers 不是传统代码库，而是一套**面向 AI 编程代理的技能框架**。它由结构化的 Markdown 配置和指令文件组成，指导 Claude Code、Codex、Cursor 等 AI Agent 按照软件工程最佳实践来工作。

核心能力：

- **可组合技能系统**：技能像乐高积木一样组合，覆盖设计、编码、测试、审查全流程
- **TDD 强制执行**：Agent 写实现代码前**必须先写测试**，测试不过不许提交
- **反馈闭环**：代码审查结果自动回灌到执行循环，持续改进
- **多代理编排**：大任务自动拆解，分配给专用子代理并行处理

## 为什么值得关注

GitHub Copilot 在 2026 年 6 月改为按 Token 计费后，开发者开始认真对待 AI 编程的**效率和质量**问题。Superpowers 的核心洞察是：**AI 编程的问题不在于"能不能写代码"，而在于"能不能按工程规范写代码"**。

这个项目完全由 Markdown 配置文件驱动，不包含传统可执行代码。这也印证了一个趋势：**"提示词工程 + 配置设计"已成为 AI 时代的一等软件资产**。

## 怎么用

**第一步：安装**

```bash
# 克隆到本地
git clone https://github.com/obra/superpowers.git

# 将技能文件复制到你的项目根目录
cp -r superpowers/skills/ your-project/.skills/
```

**第二步：在 Claude Code / Cursor 中加载**

如果你用 Claude Code，启动时指定技能目录：

```bash
claude --skills ./your-project/.skills/
```

如果你用 Cursor 或其他编辑器，只需把 `.skills/` 目录放在项目根目录即可，AI Agent 会自动读取其中的指令文件。

**第三步：使用**

加载后，你的 AI 编程助手会自动遵循以下工作流：

1. **需求分析**：先理解需求，输出技术设计文档
2. **TDD 循环**：写测试 → 写实现 → 跑测试 → 重构
3. **代码审查**：自动检查代码质量、安全性和一致性
4. **反馈改进**：根据审查结果修正代码

你只需要正常给 AI 下需求，它会自动按这套流程执行。

## 实际效果

根据项目方公布的数据和社区反馈，使用 Superpowers 前后对比如下：

| 指标 | 无 Superpowers | 有 Superpowers |
|------|--------------|----------------|
| 首次提交测试通过率 | 约 60% | 约 90%+ |
| 需要人工重构的比例 | 高 | 明显降低 |
| 第一次提交质量 | 可用但不规范 | 接近资深工程师 |
| Token 消耗 | 基准 | 略高（多了测试和审查步骤） |

> **小贴士**：Token 消耗会略高（约 10-15%），但省下的**人工审查和重构时间远超这部分成本**。

## 进阶提示

- **自定义技能**：在 `.skills/` 目录下创建自己的 Markdown 技能文件，可以定制团队专属的编码规范和工作流
- **渐进式引入**：不用一次性加载全部技能，按项目需要只复制对应技能文件即可
- **版本管理**：建议把 `.skills/` 纳入 Git 仓库，随项目一起迭代，团队共享同一套规范

## 举一反三

- **非代码项目**：技能框架不只适用于编程，可以仿照它的思路，给 AI 写作助手、AI 设计工具也配上"方法论"配置文件
- **教学场景**：把 Superpowers 作为案例，帮助团队理解"AI 不需要你告诉它怎么写代码，而是需要你告诉它怎么想问题"
- **竞品搭配**：配合其他 AI 编程工具的插件系统使用，在不同 Agent 间统一工程规范
