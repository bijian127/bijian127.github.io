**时间**：周四 20:00 | **字数**：2000-2500字 | **形式**：深度长文 + 代码截图

**场景**：你们公司的文档散落在飞书、钉钉、NAS、邮件附件各处。新人入职问个问题要跑 3 个人。

**效果预览**：
- Before：新人问"报销流程是什么"，老员工说"看飞书文档"，新人找了 20 分钟没找到
- After：新人问 AI"报销流程是什么"，3 秒出答案，附带文档链接和操作步骤

**操作步骤**：
1. 部署 Dify（GitHub 136K Star，开源 AI 应用平台，支持 Docker 一键部署）
2. 创建知识库，上传所有公司文档（PDF/Word/Markdown/网页）
3. 配置 RAG 检索策略，选择合适的分块和向量化方案
4. 发布为 Web 应用或 API，嵌入飞书/钉钉机器人

```
Dify 知识库配置要点：

1. 文档分块策略：
   - 分块大小：500-1000字
   - 重叠：100-200字
   - 适合大多数公司文档

2. 检索策略：
   - 混合检索（向量+关键词）
   - Top K：5（返回最相关的5个片段）
   - 相似度阈值：0.7

3. 系统提示词：
   "你是公司内部知识库助手。根据检索到的文档内容回答问题。
   如果文档中没有相关信息，明确说'文档中未找到相关信息，建议咨询XX部门'。
   回答时附上信息来源的文档名称。"
```

**进阶**：Dify v1.13 新增了 Human-in-the-Loop 功能，可以在 AI 回答前加入人工审核环节，确保准确性。
